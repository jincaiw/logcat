# v1.5.4 更新日志

## 发布日期

2026-04-22

## Bug 修复

### 1. 白名单功能修复

- **问题**：白名单功能在代码中存在调用嵌套问题
- **影响**：所有推送平台（钉钉、飞书、企业微信、邮箱、Syslog）的白名单配置
- **修复**：将白名单检查集成到日志处理流程中，日志入库后先检查白名单

### 2. 未匹配日志丢弃功能修复

- **问题**：`DropUnmatched` 字段在数据库中存在调用问题
- **影响**：配置"未匹配日志丢弃"选项后不生效
- **修复**：实现未匹配日志时的丢弃逻辑

### 3. 白名单列表显示优化

- **问题**：白名单列表不显示匹配的字段名称，用户无法确认配置是否正确
- **修复**：在白名单列表中新增"匹配字段"列，显示对应的字段名称

### 4. 日志查看页面优化

- 筛选状态显示中文（pending→待处理、matched→已匹配、whitelisted→白名单等）
- 日志列表列宽支持拖拽调整
- 长文本自动截断显示

## 功能说明

### 日志筛选状态

| 状态          | 含义         | 是否推送 |
| ----------- | ---------- | ---- |
| pending     | 待处理        | -    |
| matched     | 策略匹配成功     | ✅    |
| discarded   | 策略匹配且动作=丢弃 | ❌    |
| whitelisted | 白名单匹配      | ❌    |
| unmatched   | 未匹配任何策略    | ❌    |

### 处理流程

```
日志入库（状态: pending）
    ↓
遍历所有策略
    ↓
白名单检查 → 匹配则跳过当前策略
    ↓
筛选条件检查 → 匹配则停止遍历
    ↓
推送告警 / 丢弃日志
```

1. **备份数据库**：升级前建议备份 `syslog2bot.db`
2. **重新测试白名单**：升级后请重新测试白名单配置是否生效
3. **检查日志状态**：升级后在日志查看页面确认筛选状态显示正确
